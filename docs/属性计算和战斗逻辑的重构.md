# 属性计算和战斗逻辑的重构

昨天一上课，我打开一周没动的RPG项目，本来是想要完善一下经验系统，但是一想到升级就想到了属性提升，毕竟属性不提升那升级有什么用？然后我试图实现六维和自由属性点，自然又想到了下四：生命成长、法力成长、速度、幸运，然后又想到了更多二级属性：格挡率、闪避值、命中值$\dots$这一想就停不下来了，于是干脆开始着手实现生物的属性和相关战斗数据的计算。

## 属性计算

首先开了一个`definition.hpp`用来存储相关定义，然后开了几个结构体。

`Attribute`:上六维和下四维，表示基本属性。

```cpp
struct Attributes
{                    // 基本属性
    int strength;    // 力量
    int physique;    // 体质
    int dexterity;   // 灵巧
    int esthesia;    // 感知
    int bewitchment; // 魔力
    int willpower;   // 意志

    int life_growth;  // 生命成长
    int magic_growth; // 法力成长
    int speed;        // 速度
    int luck;         // 幸运
};
```

`DerivedStats`:表示次生属性，比如最大生命值、最大法力值、基础物理攻击力、法术命中值、真实减伤$\dots$等等，只能说RPG玩得太多了，一联想就停不下来，后面也因为初始的过度设计吃了很多苦头。

```cpp
struct DerivedStats
{
    int max_hp;                       // 最大生命值
    int max_mp;                       // 最大法力值

    int magical_hit_value;            // 法术命中值

    double magical_crit_rate;         // 魔法暴击率
    double true_crit_rate;            // 真实暴击率
    int defense;                      // 防御力
    double physical_damage_reduction; // 物理减伤
    double magical_damage_reduction;  // 魔法减伤
    double true_damage_reduction;     // 真实减伤
    double ignore_defense_rate;       // 物理无视防御几率

    double magical_damage_increase;   // 魔法增伤

```

`BonusStats`:表示增益属性，就代表那些装备和buff带来的加成，其中有很多属性是既有百分比加成又有固定数值加成，所以我直接开了一个`Mod`结构体用来存储同一个属性对应的不同类型加成，并重载了运算符+,+=

```cpp
struct Mod
{
    double pct = 0.0;
    int flat = 0;

    void clear()
    {
        pct = 0.0;
        flat = 0;
    }

    Mod &operator+=(const Mod &other)
    {
        pct += other.pct;
        flat += other.flat;

        return *this;
    }

    Mod operator+(const Mod &other)
    {
        return {this->pct + other.pct, this->flat + other.flat};
    }
};

struct BonusStats
{
    void clear()
    {
        *this = {};
    }

    Mod strength, physique, dexterity, esthesia, bewitchment, willpower;
    Mod life_growth, magic_growth, speed, luck;
    Mod max_hp, max_mp, base_physical_attack_power, base_magical_attack_power, base_true_attack_power;
    Mod evasion_value, physical_hit_value, magical_hit_value;
    Mod defense;
    Mod physical_damage, magical_damage;

    double physical_critical_damage_bonus;  // 物理爆伤加成
    double magical_critical_damage_bonus;   // 魔法爆伤加成
    double true_critical_damage_bonus;      // 真实爆伤加成
    double physical_crit_rate_bonus;        // 物理暴击率加成
    double magical_crit_rate_bonus;         // 魔法暴击率加成
    double true_crit_rate_bonus;            // 真实暴击率加成
    double block_rate_bonus;                // 格挡率加成
    double physical_damage_reduction_bonus; // 物理减伤加成
    double magical_damage_reduction_bonus;  // 魔法减伤加成
    double true_damage_reduction_bonus;     // 真实减伤加成
    double ignore_defense_rate_bonus;       // 物理无视防御几率加成
    double physical_damage_increase_bonus;  // 物理增伤加成
    double magical_damage_increase_bonus;   // 魔法增伤加成
};
```

有了这么多属性，自然需要引入属性计算，这里我就随便给了点计算公式，比如

```cpp
double base_mp = (f.esthesia * ATTR_MP_PER_ESTHESIA) +
                     (f.bewitchment * ATTR_MP_PER_BEWITCHMENT) +
                     (f.willpower * ATTR_MP_PER_WILLPOWER) +
                     (f.magic_growth * level * GROWTH_MP_PER_MAGIC_GROWTH_PER_LEVEL) +
                     race_mp;

return apply_standard(base_mp, bonus.max_mp.pct, bonus.max_mp.flat);
```

那些参数就塞到了`config.hpp`里。且对于这么多的计算函数，我开了一个`stat_calculator.hpp`并使用了一个计算类`StatCalculator`来实现计算函数。

搞定了属性的计算，接下来就需要同步到`Creature`类中，将`Attribute`等属性结构体导入。

```cpp
Attributes attrs;
DerivedStats stats;
```

现在我们的“生物”只是一个具有这些属性的空壳子，要实现角色穿脱装备，或者升级，甚至是怪物受到什么减速效果，我们都需要及时**同步状态**。对于具体的同步逻辑，是否需要考虑多进程设计，暂且按下不表，我们首先需要实现的是对具体属性的更新。

所以我在`Creature`类中实现了一个`refresh_stats`函数，用来更新状态。具体流程大概是：

$$
清空原有装备和\text{buff}等加成 \\
 \Downarrow \\
 重新计算累加所有装备和\text{buff}加成 \\
 \Downarrow \\
 重新计算基本属性 \\
 \Downarrow \\
 重新计算次生属性 \\
 \Downarrow \\
 属性对齐（比如最大血量变化后现有血量按照比例变化） \\
 \Downarrow \\
 更新脏标记为否
$$

其中脏标记是用于优化性能的，虽然来说以当前项目的体量根本没有必要优化。具体原理类似于线段树的懒标记，核心就是**真正需要的时候再更新**。

例如当前我穿上了一件装备，但是我接下来既没有看属性面板也没有发起战斗，那此时我们就将这个更新延迟，拖下来，并且将脏标记置为真，表明当下次需要获取当前属性值的时候，我就需要更新了。

同时，在“重新计算基本属性”这步，我把它和“重新计算次生属性”的函数放在一块了，并用一个`FinalAttributes`类存储我们**最终的**基本属性。为什么这样做？因为在我的定义里，`attrs`仅应当用于存储那些**永恒的**，**不变的**基本属性。比如加点带来的属性（也不一定？或许我以后需要加入死亡后经验惩罚，掉级之后由自由属性点带来的加成会被回退），比如角色本身就具有的属性。

总之，我使用了下面的函数来计算最终基本属性：

```cpp
FinalAttributes StatCalculator::make_final_attrs(const Attributes &attr, const BonusStats &bonus)
{
    return {
        apply_standard(attr.strength, bonus.strength.pct, bonus.strength.flat),
        apply_standard(attr.physique, bonus.physique.pct, bonus.physique.flat),
        apply_standard(attr.dexterity, bonus.dexterity.pct, bonus.dexterity.flat),
        apply_standard(attr.esthesia, bonus.esthesia.pct, bonus.esthesia.flat),
        apply_standard(attr.bewitchment, bonus.bewitchment.pct, bonus.bewitchment.flat),
        apply_standard(attr.willpower, bonus.willpower.pct, bonus.willpower.flat),
        apply_standard(attr.life_growth, bonus.life_growth.pct, bonus.life_growth.flat),
        apply_standard(attr.magic_growth, bonus.magic_growth.pct, bonus.magic_growth.flat),
        apply_standard(attr.speed, bonus.speed.pct, bonus.speed.flat),
        apply_standard(attr.luck, bonus.luck.pct, bonus.luck.flat)};
}
```

然后在次生属性计算前，先存一个快照：

```cpp
const FinalAttributes f_attrs = StatCalculator::make_final_attrs(attrs, bonus);
```

接下来再调用计算函数进行次生属性的计算，比如：

```cpp
stats.max_hp = StatCalculator::calc_max_hp(f_attrs, level, bonus, races.init_hp);
```

## 战斗逻辑

原来的战斗逻辑实在是太过简陋，我们需要定义新的战斗逻辑。

首先是攻击。显然，攻击有三个不能忽视的关键元素：攻击者、被攻击者、攻击方式。

对于攻击者和被攻击者，我们直接传入两个`Creature&`引用。然而攻击方式呢？这时候我们就又要引入新的类型了：`SkillData`

我们最终想要实现的是：一道攻击，只由这个攻击本身的种类来决定性质，比如它的名字，描述，伤害倍率，伤害类型$\dots$，而攻击者只需要调用。

所以我们需要创建一个技能类，使用时，由**攻击者**发出指令，调用技能释放函数`execute_skill`，并传入角色属性、技能种类，然后在对应执行函数中，由技能本身的计算伤害方式，结算物伤、法伤、真伤，并计算“特殊效果”强度，随后以结构体类型返回，并执行进一步的爆伤逻辑。

在这里，我们采用“id调用”的方式，也即每个生物实例持有的是技能id列表，当需要调用时，传入id，由技能类本身判断是什么技能，从而进一步计算。

现在我们需要定义技能类型。

```cpp
enum class DamageType
{
    Physical,
    Magical,
    True
};

enum class TargetType
{
    Self,
    SingleEnemy,
    MultiEnemies,
    SingleAlly,
    MultiAllies
};

struct SkillData
{
    int id = 0;
    std::string skill_name;
    std::string description;

    int cost_hp, cost_mp, cost_sp;  //消耗
    int cooldown;   //冷却

    TargetType target_type;     //技能目标
    double hit_rate_bonus = 0.0;  //命中修正
    double crit_rate_bonus = 0.0; //暴击修正

    struct DamageInfo {
        DamageType type;
        double multiplier;
        int fixed_bonus;
    };
    std::vector<DamageInfo> damages;

    struct EffectTrigger {
        int effect_id;
        double chance;
        int power;
    };
    std::vector<EffectTrigger> effect_triggers;
};
```

现在，我们是时候开始考虑数据导入了！因为我们肯定不可能在代码中直接写：`name = "普通攻击"`,`cost_mp = 10`这样的定义，那样维护起来太不方便且低效了，**数据归数据，逻辑归逻辑**。所以我们需要考虑引入并读取数据文件来处理。

### 数据引入

在引入数据前，我们首先必要去做的就是完成对数据类型的定义。比如说一个种族，它应该需要我们导入什么属性和数据，而技能，又应该有什么数据。

#### 种族&职业

在这里我采用了一个比较传统的设计，也是我本身比较喜欢的一个设计：种族+职业。 ~~可能是因为网游文看多了~~

比如说人类战士，亡灵法师，精灵弓箭手...这样的。

首先我们考虑，种族和职业能给生物带来什么属性：譬如说人类的各方面较为中庸，兽人的力量和体质非常强悍，但似乎天生难以沟通魔力...所以我们引入`base_attr`属性，用于描述种族和职业所具有的天生属性：

```json
"base_attr": {
      "strength": 12,
      "physique": 10,
      "dexterity": 15,
      "esthesia": 8,
      "bewitchment": 5,
      "willpower": 8,
      "life_growth": 120,
      "magic_growth": 80,
      "speed": 110,
      "luck": 20
    }
```

其次，我为其赋予了初始的生命值、法力值、精力值：

```json
"init_hp": 150,
"init_mp": 60,
"init_sp": 100
```

最后则是最能体现种族和职业特性的东西：天赋技能。

比如说战士会旋风斩，亡灵可以释放恐惧技能，精灵的蓝量恢复比较快...

```json
"race_skills": [5001]
"job_skills": [5101, 5102]
```

再加上通用的`id`,`name`,`description`，就构成了这两种相似度较高的数据。

#### 技能

这是战斗系统的核心，我们现在来定义一个技能应当具有什么属性。

- 在除了传统的`description`字段外，我们还需要一个详细的描述以体现技能效果，譬如技能“火球术”对敌人造成120%法术伤害，且有30%概率触发持续时间为5回合的灼烧效果。
  
  所以我们有`display_effect`属性：

  ```json
  "display_effect": "消耗自身20%生命值，对对方造成200%法术攻击的真实伤害。"
  ```

- 下面就是释放技能所需要消耗的代价：血量、蓝量，或者是精力，甚至可能是以后的其他的奇奇怪怪的属性。写到这里我突然想到，对于我目前的这种设计：

  ```json
  "cost_hp": 0,
  "cost_mp": 10,
  "cost_sp": 0
  ```

  可以看出，完全无法处理类似于百分比消耗等更加复杂的逻辑，如果以后想要增加更多属性的消耗，比如亡灵能量，很难去维护，所以在思考过后，我采用了如下设计：

  首先在`base_type.hpp`中定义一个枚举类型：

  ```cpp
  enum class StatusType  // 状态类型
  {
      HP,
      MP,
      SP,
      EXP
  };
  ```

  然后在`skill_type.hpp`中同样定义一个枚举类型：

  ```cpp
  enum class CostMethod  // 消耗类型
  {
      FLAT,         // 固定值
      PERCENT_MAX,  // 最大百分比
      PERCENT_CUR   // 当前百分比
  };
  ```

  最后，在`skill_data.hpp`中将原来的`cost_hp`,`cost_mp`,`cost_sp`替换为如下字段：

  ```cpp
  struct Costs
  {
    StatusType status_type;
    CostMethod cost_method;
    double value;
  };
  std::vector<Costs> cost;
  ```

  这样的话，对于技能的消耗，就定义完成了，以后想要增减属性也非常容易。

- 冷却：其实没什么好说的，直接设定一个初始值，应用在具体对象上时再根据加成进行冷却缩减。

- 作用目标：定义为单个敌人，多个敌人，单个友方，多个友方等
